# Changelog

All notable changes to UmbraVOX are documented in this file.

The format is based on Keep a Changelog 1.1.0 and this project aims to follow Semantic Versioning.

## [Unreleased]

### Added
- Added NixOS QEMU VM image (`nix build .#vm-image`) for fully isolated build/test/release execution with zero external dependencies.
- Added Haskell VM smoke orchestration (`umbravox vm-smoke`) with image caching, source disk creation via genext2fs, and QEMU/KVM boot management.
- Added `make vm-smoke`, `make vm-image-build`, `make vm-image-clean`, and `make image-clean` targets for VM-isolated pipeline execution.
- Added `genext2fs` to nix-shell and flake devShell for non-root source disk image creation.
- Added in-guest smoke pipeline script (`scripts/vm-smoke-run.sh`) that runs build, test, verify, complexity, license, format-check, and release-linux inside the isolated VM.
- Added app-layer AEAD encryption module (`UmbraVox.Storage.Encryption`) providing AES-256-GCM field-level encryption with HKDF-derived per-identity storage keys, migration-safe plaintext passthrough, and deterministic test-key injection.
- Added fail-closed persistence restore behavior: conversations that fail to decrypt are skipped with a `persistence.decrypt.failed` log event instead of crashing the application.
- Added persistence encryption model documentation (`doc/persistence-model.md`) covering encrypted vs. plaintext fields, residual metadata exposure, and rationale.
- Added bounded MVP assurance statement and post-MVP target statement to the assurance roadmap, with reconciled wording across README, AGENTS.md, and the assurance matrix.
- Added persistence confidentiality row to the assurance matrix evidence ledger.
- Added release gate script (`scripts/release-gate-assurance.sh`) requiring the assurance matrix to be present, complete, and not stale relative to crypto source changes.
- Added `make release-gate-assurance` target for assurance matrix freshness checking.
- Added Haskell orchestration subcommands (`runOrchestratedBuild`, `runOrchestratedTest`, `runOrchestratedVerify`) with stable exit-code mapping (0=success, 1=failure, 124=timeout, 127=missing tool) behind `--orchestrated` flag.
- Added runtime logging regression tests (`Test.App.RuntimeLog`) covering redaction completeness, list ordering, permission enforcement, and sensitive-field coverage.
- Added storage encryption tests (`Test.Storage.Encryption`) covering round-trip, migration-safe passthrough, wrong-key rejection, empty/large content, and key derivation determinism.
- Added opt-in `build-haskell`, `test-haskell`, and `verify-haskell` bridge wrappers that keep the legacy `Makefile` path by default while documenting the future Haskell orchestration switch.
- Added explicit active TODO tasks to keep `Makefile` as a thin wrapper while moving build/release orchestration logic into Haskell entrypoints with `nix-shell` workflow parity.
- Added an opt-in Haskell readiness bridge (`make release-lane-readiness-haskell`) that shells out to the existing aggregate readiness script while keeping the shell bridge as the source of truth for now.
- Added a phased release-orchestration migration plan that keeps the current shell bridge honest while tracking the move to Haskell entrypoints.
- Added an experimental AppImage scaffold target plus a non-authoritative smoke placeholder that validates layout only.
- Added consolidated tracking hygiene so completed release-lane and readiness-scaffold work is moved out of `TODO.txt` and only active items remain there.
- Added placeholder compliance gates for future SBOM and license-bundle work (`make release-compliance`, `make release-sbom`, `make release-license-bundle`) that fail clearly when required tools are absent and emit no artifacts yet.
- Added explicit release-target packaging workflow with Linux bundle and source-release targets for Windows CLI, macOS terminal, BSD terminal, and FreeDOS research packaging.
- Added tiered test harness entrypoints (`required`, `core`, `tcp`, `fault`, `recovery`, `soak`, `deferred`) and expanded deterministic suite segmentation.
- Added runtime troubleshooting logging toggles, metadata-focused event emission, and associated regression coverage.
- Added generated/codegen wiring for preserved Haskell wrappers and generated FFI/C bridge participation in active build/test graphs.
- Added transport fallback progress/status reporting and expanded network hardening assertions for fallback endpoint behavior.
- Added release-lane microVM smoke scaffolding for isolated validation of produced Linux artifacts.
- Added documented microVM smoke execution hooks for host-provided QEMU/Firecracker runner commands, plus pinned-input entrypoints for QEMU and Firecracker smoke invocation.
- Added Firecracker pinned-boot smoke documentation as a completed invocation path, while keeping maintained guest-image/config ownership and default in-guest verification out of scope for now.
- Added explicit platform-lane posture and cross-target policy documentation covering native-lane placeholders, source-release targets, and non-authoritative cross-built outputs.
- Added aggregate release-readiness documentation covering host quality gates, Linux smoke coverage, microVM prerequisite checks, and the remaining authoritative-lane gaps.
- Refined release-lane gap output so Firecracker guest-image/config evidence, native runner evidence, and lane-specific next actions point at the exact remaining tasks.
- Tightened the microVM scaffold so pinned QEMU/Firecracker runs require an explicit in-guest verification command template before they proceed.

### Changed
- Updated flake.nix to expose vm-image package on x86_64-linux and added QEMU/Firecracker/genext2fs to devTools.
- Expanded runtime logging redaction list from 7 to 15 fields, adding content, answer, key, secret, passphrase, password, token, and session_id.
- Hardened runtime log file permissions to enforce 0600 on every write (not just creation) with single-writer PID tracking.
- Updated Haskell orchestration bridge targets (`build-haskell`, `test-haskell`, `verify-haskell`) to pass `--orchestrated` flag when `UMBRAVOX_USE_HASKELL_ORCH=1` is set, routing to real Haskell implementations instead of shelling back to Make.
- Clarified the AppImage track as scaffold-only and kept it outside the authoritative release posture.
- Reworked messaging MVP hardening flow into explicit fast-gate and targeted suite runs with artifact logging.
- Updated Linux release launch mechanics to use bundled-loader invocation instead of invalid interpreter patching assumptions.
- Improved TUI interaction model with richer simulation coverage for dialogs, menus, browse pagination/search, and runtime status flows.
- Updated release documentation and tracking to reflect the new placeholder SBOM/license compliance gates without overstating artifact generation.
- Clarified release docs to distinguish working Linux container smoke, completed QEMU/Firecracker pinned-boot invocation paths, and the still-missing maintained guest-image/config and authoritative in-guest release execution.
- Updated release tracking to mark only implemented documentation/scaffold tasks complete for platform lanes and cross-target policy, while leaving native execution gaps open.
- Updated quickstart/release lane guidance to state the current aggregate readiness checks and keep unfinished native-lane, parity, and in-guest execution gaps explicit.
- Updated microVM lane guidance to call out the explicit in-guest verification command template now required for pinned runs.

### Fixed
- Fixed host-only peer connect flow to try default port lists with clearer diagnostics and endpoint reporting.
- Fixed non-deterministic test-run behavior by adding explicit fast-gate timeout control and timeout failure reporting in `make test`.
- Fixed restoration/send safety by preventing disconnected restored sessions from being treated as live-sendable peers without reconnect.

### Security
- Added app-layer AEAD storage encryption module for persistent message content and conversation name confidentiality at rest (AES-256-GCM with HKDF-derived keys).
- Strengthened session restore safety boundaries to avoid local loopback/self-delivery risk after persistence restore.
- Tightened logging assurance posture through redaction/scope controls and separation of pre-auth transport arrival vs trusted session establishment semantics.
- Restored full green formal verification gate (17 F* specs) and retained verification as a hard quality gate.
