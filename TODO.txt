--- 0b: F* Verification (DESIGN REQUIREMENT: all crypto must have F* proofs) ---

NOTE: F* formal verification is a MANDATORY design requirement for every
cryptographic primitive. No primitive is considered complete without:
  1. F* specification matching the NIST/RFC standard
  2. KAT vector lemmas (assumed, validated by Haskell test suite)
  3. Structural correctness lemmas (output length, roundtrip, etc.)
  4. Correspondence proof: F* spec ↔ Haskell implementation

Current F* status (17 total, all verified):
  [x] Spec.SHA256.fst       (FIPS 180-4)    — 3 KAT vectors, structural lemmas
  [x] Spec.SHA512.fst       (FIPS 180-4)    — 2 KAT vectors, structural lemmas
  [x] Spec.HMAC.fst         (RFC 2104/4231) — 2 KAT vectors, PRF axiom
  [x] Spec.HKDF.fst         (RFC 5869)      — 1 KAT vector, length guarantee
  [x] Spec.AES256.fst       (FIPS 197)      — 1 KAT vector, roundtrip lemma
  [x] Spec.GaloisField.fst  (SP 800-38D)    — algebraic properties
  [x] Spec.GCM.fst          (SP 800-38D)    — 2 KAT vectors, roundtrip + tampering
  [x] Spec.ChaCha20.fst     (RFC 8439)      — 2 KAT vectors (partial: assumes double_round)
