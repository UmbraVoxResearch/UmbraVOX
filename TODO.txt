[x] BUG-001 invNTT butterfly sign error in ML-KEM-768 (CRITICAL — FIXED)
  FIPS 203 Algorithm 10: f[j+len] = zeta*(f[j+len]-f[j])
  Was incorrectly: zeta*(f[j]-f[j+len]). Masked by SHA-256 substitutes,
  exposed by SHA-3/SHAKE integration. Fixed in commit 5ab1f89.

[x] BUG-002 HKDF-Expand dead code condition (FIXED)
  hkdfExpand line 60: BS.length (BS.concat []) >= 0 && counter > n
  Nonsensical always-true clause. Simplified to: counter > n.

[x] BUG-003 GCM incr32 counter wrap at 2^32 (FIXED)
  Counter used Word64 arithmetic without 32-bit mask.
  Added .&. 0xffffffff to match SP 800-38D 32-bit counter semantics.

[ ] BUG-004 Double Ratchet: use Maybe for uninitialized rsDHRecv
  Currently uses BS.empty sentinel. Should use Maybe ByteString
  for type safety. File: Signal/DoubleRatchet.hs line 163.

[ ] BUG-005 Double Ratchet: skipMessageKeys eviction policy
  Evicts only one key when map exceeds maxSkip (1000).
  Should implement "older than 500 steps" eviction per spec.
  File: Signal/DoubleRatchet.hs lines 314-317.

[ ] BUG-006 Consensus Block.hs: incomplete BlockHeader
  Missing 6 required fields: bhBlockNo, bhIssuerVK, bhVRFProof,
  bhVRFOutput, bhSignature, bhHeartbeat. File: Consensus/Block.hs.

[ ] BUG-007 Consensus LeaderElection.hs: wrong type signature
  Takes ByteString -> Word64 -> Bool. Should take stake ratio,
  VRF output, and active slot coefficient. Missing 128-bit
  fixed-point (64.64) arithmetic for threshold computation.
