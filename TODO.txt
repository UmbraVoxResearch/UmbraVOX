--- 0b: F* Verification (DESIGN REQUIREMENT: all crypto must have F* proofs) ---

cryptographic primitive. No primitive is considered complete without:
  1. F* specification matching the NIST/RFC standard
  2. KAT vector lemmas (assumed, validated by Haskell test suite)
  3. Structural correctness lemmas (output length, roundtrip, etc.)
  4. An explicit correspondence/refinement story between the F* model and the
     active Haskell implementation

BUGS FOUND BY 11-AGENT EXPERT CRYPTO REVIEW #2 (2026-04-24)
================================================================================

--- Fixed ---

[x] BUG-011 ML-KEM SampleNTT: finite XOF buffer with zero-padding fallback
  FIPS 203 requires XOF to produce bytes indefinitely. Zero-padding produces
  wrong polynomials. Fix: error on exhaustion (astronomically unlikely with
  672+ bytes), never zero-pad.

[x] BUG-012 ML-KEM matrix A: XOF index ordering (j,i) vs FIPS 203 (i,j)
  xof rho j i produces rho||j||i. FIPS 203 Algorithm 12 says A[i,j] = XOF(rho,i,j).
  Self-consistent but breaks FIPS 203 KAT vector compatibility.
  Fix: swap to xof rho i j in both kpkeKeyGen and kpkeEncrypt.

[x] BUG-013 ML-KEM decaps: non-constant-time ciphertext comparison
  `ct' == ct` is variable-time ByteString equality on 1088-byte ciphertext.
  FIPS 203 Section 7.3 requires constant-time selection to prevent
  implicit-rejection oracle attacks.
  Fix: use constantEq pattern (fold with XOR accumulator).

[x] BUG-014 Double Ratchet: nonce construction wastes 32 bits
  Last 4 bytes of 12-byte GCM nonce always zero.
  Fix: use 8 bytes HMAC + 4 bytes counter = full 12-byte nonce.

[x] BUG-015 Double Ratchet: deterministic DH key generation
  New DH keypairs derived via HMAC instead of CSPRNG.
  Fixed: replaced with randomBytes from CSPRNG (IMPL-002).
