# Vendoring fiat-crypto field arithmetic

This document records the exact steps to fetch and pin the fiat-crypto C files
into `csrc/fiat/`.

## Choosing a release tag

Always vendor from a tagged release, never from `master`. Check
https://github.com/mit-plv/fiat-crypto/releases for the latest tag.

As of initial setup the recommended tag is `v0.0.9` (adjust when updating).

## Fetch steps

```sh
# Set the tag to vendor
FIAT_TAG=v0.0.9
FIAT_BASE="https://raw.githubusercontent.com/mit-plv/fiat-crypto/${FIAT_TAG}/fiat-c/src"
DEST="csrc/fiat"

# 64-bit GF(2^255-19) — covers Curve25519 and the Ed25519 base field
curl -fsSL "${FIAT_BASE}/fiat_25519_64.c"  -o "${DEST}/fiat_25519_64.c"
curl -fsSL "${FIAT_BASE}/fiat_25519_64.h"  -o "${DEST}/fiat_25519_64.h"

# Optional: 32-bit variant for illumos/SmartOS 32-bit and older BSDs
# curl -fsSL "${FIAT_BASE}/fiat_25519_32.c" -o "${DEST}/fiat_25519_32.c"
# curl -fsSL "${FIAT_BASE}/fiat_25519_32.h" -o "${DEST}/fiat_25519_32.h"
```

> Note: some older fiat-crypto tags use `fiat_curve25519_64.c` instead of
> `fiat_25519_64.c`. If the curl above returns 404, try:
> ```sh
> curl -fsSL "${FIAT_BASE}/fiat_curve25519_64.c" -o "${DEST}/fiat_25519_64.c"
> curl -fsSL "${FIAT_BASE}/fiat_curve25519_64.h" -o "${DEST}/fiat_25519_64.h"
> ```

## Checksum pinning

After fetching, record SHA-256 checksums so future CI can verify integrity:

```sh
sha256sum csrc/fiat/fiat_25519_64.c csrc/fiat/fiat_25519_64.h \
    | tee csrc/fiat/SHA256SUMS
```

Commit `SHA256SUMS` alongside the vendored files.

## Fetch the upstream license

```sh
curl -fsSL "https://raw.githubusercontent.com/mit-plv/fiat-crypto/${FIAT_TAG}/LICENSE" \
    -o csrc/fiat/LICENSE.fiat-crypto
```

The fiat-crypto license is MIT. Reproducing it here satisfies the attribution
requirement for static linking.

## Verifying correctness of the fetch

The files should contain the fiat-crypto copyright header near the top.
Spot-check with:

```sh
head -5 csrc/fiat/fiat_25519_64.c
# Expected: /* Autogenerated: ... */  or similar fiat-crypto preamble
```

## Build integration

After vendoring, add the C files to the Cabal `c-sources` stanza in
`UmbraVox.cabal` under the `pure-haskell` flag guard:

```cabal
  if !flag(pure-haskell)
    c-sources:
      csrc/fiat/fiat_25519_64.c
    include-dirs: csrc/fiat
```

No changes to `csrc/` include paths are needed; `csrc` is already in
`include-dirs`.

## Updating to a new tag

1. Update `FIAT_TAG` above.
2. Re-run the fetch steps.
3. Re-run the checksum step and commit updated `SHA256SUMS`.
4. Rebuild and re-run the test suite (`./uv test`).
5. Update the pinned tag in this document.
